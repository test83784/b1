Vazamento WebRTC

Criado em02-10-2022|Atualizado em20-10-2022|Informática
|Visualizações:

Houve vários casos confirmados em que o endereço IP dos usuários foram vazados através de uma função chamada WebRTC, mesmo que uma VPN esteja sendo usada. O WebRTC está embutido nos navegadores que normalmente usamos, como o Chrome e o Firefox, tornando-o um dos problemas que ameaçam nossa privacidade.

O que é WebRTC?

WebRTC significa ‘Comunicação em Tempo Real’ e é um padrão que permite o bate-papo por vídeo, áudio, transferência de arquivos e streaming ao vivo entre navegadores.

A tecnologia é de código aberto e está atualmente implementada nos principais navegadores como o Chrome, FireFox, Safari, Opera e Edge.

Uma das principais características do método de comunicação é que ele usa o método P2P (peer-to-peer) de comunicação direta entre terminais, semelhante ao Torrent. Métodos convencionais de comunicação, tais como WebSocket, comunicam-se através de um servidor central. Com o WebRTC, a troca de dados ocorre entre os usuários. Isto evita baixa latência e lentidão e reduz a carga sobre o servidor.

Mecanismo de vazamento WebRTC

Embora esteja se tornando cada vez mais popular, grandes preocupações com privacidade têm sido descobertas.

Mesmo quando se utiliza uma VPN, o endereço IP da fonte de conexão pode ser passado a um terceiro. Este problema é conhecido como vazamento WebRTC ou WebRTC leak.

Devido ao método de comunicação P2P, a comunicação só pode ser estabelecida quando ambas as partes conhecem o endereço IP uma da outra, assim como não se pode fazer uma chamada telefônica sem conhecer o número de telefone da outra.

Serviços e websites que utilizam WebRTC podem utilizar o WebRTC para conhecer o verdadeiro endereço IP do usuário (não o IP da VPN, mas o IP público da fonte de conexão). Este IP público é uma informação pessoal, como um endereço residencial, e se a outra parte souber, eles podem te identificar e rastrear.

Por que ocorre o vazamento de IP?

A razão pela qual WebRTC vaza o endereço IP mesmo quando o usuário está em uma VPN reside no servidor STUN usado para estabelecer a comunicação P2P: Uma VPN usa um servidor STUN para converter alternadamente o IP Real (o IP público da fonte) e o IP do servidor VPN.

Uma tabela IP é criada no servidor STUN contendo tanto o IP real quanto o servidor VPN, que pode ser lida por terceiros que fornecem serviços ou websites usando WebRTC.

Desta forma, o endereço IP local pode ser vazado, mesmo quando se utiliza uma VPN.

Como verificar vazamentos no WebRTC

Primeiro, verifique o IP público que você está usando atualmente sem uma VPN.

Depois, conete-se a uma VPN e acesse https://browserleaks.com/webrtc.

Se um IP diferente do IP público que você verificou primeiro for exibido, ou se a mensagem “WebRTC não está disponível” for exibida como mostrado abaixo, você não precisa se preocupar com vazamento de IP.

Os navegadores que melhor suportam proteção contra vazamentos WebRTC são Firefox, Opera, Safari e Brave, enquanto o Google Chrome e Microsoft Edge parecem ter problemas.

Para o iOS, não parece haver problema com vazamento do WebRTC desde que você esteja usando o iOS 12 ou posterior e o Safari.

Configurando o navegador para evitar vazamentos no WebRTC

Restringir ou desativar o WebRTC pode tornar alguns serviços web inutilizáveis.

Google Chrome (PC)

Para o Chrome, você pode evitar o vazamento do WebRTC instalando plug-ins.

Google Chrome (Android)

Escreva:

chrome://flags/#disable-webrtc

Encontre o item “WebRTC Stun origin header” e mude-o para “Desabilitado”.

Firefox (PC e Android)

Digite “about:config” na barra de endereço do navegador. Clique em “Use por sua própria conta e risco”.

Copie e cole “media.peerconnection.enabled” no campo de busca.

Clique duas vezes em Verdadeiro e mude para Falso.

Safari

Preferências > Avançado > Ativar o menu “Desenvolvedor”

Como o menu “Desenvolvedor” aparecendo, desmarque “WebRTC” → “Enable Legacy WebRTC API”.

Brave

O Brave pode evitar vazamentos do WebRTC de duas maneiras.

  1. Vá para Configurações e selecione ‘Brave Shield’ → ‘Fingerprint Protection’ → ‘Block all fingerprints’.

  2. Nas configurações avançadas, selecione ‘Privacidade e segurança’ → ‘WebRTC IP address handling policy’ → ‘Disable non-proxied UDP’.

Ópera

Nas configurações avançadas, altere a política de tratamento de IP para “Disable non-proxied UDP”.

Conclusão

Esta questão de vazamento do WebRTC é uma questão bastante nova de proteção à privacidade.

Os navegadores foram identificados como uma das principais causas de vazamentos de privacidade, e se você não quiser vazar elementos que te identifiquem, recomendamos evitar ao máximo o Chrome, IE e Edge e usar um navegador como o Firefox ou Brave.

VPNs gratuitas podem correr o risco de vazar o IP. Além de usar os métodos acima, é essencial usar um serviço VPN que ofereça proteção contra vazamentos de WebRTC.
